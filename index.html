Gestor de Gastos Mensuales Personales

Este proyecto es una aplicación de gestión de gastos personales desarrollada en JavaScript. Permite al usuario registrar ingresos y gastos, clasificarlos en diferentes categorías (fijos, imprevistos y ahorro) y calcular el saldo disponible. La primera versión funciona mediante el uso de `prompt`, `alert` y `console.log` interactuando con las funciones que se ven debajo

Funciones Básicas

pedirTipo(): Pide al usuario que seleccione entre "ingreso" o "gasto"
Retorna: "ingreso" o "gasto"
pedirMonto(tipo): Pide al usuario un monto numérico mayor o igual a 0
Argumentos: tipo ("ingreso" o "gasto")
Retorna: Número ingresado
pedirDescripcion(tipo): Pregunta si desea agregar una descripción; si acepta, solicita una
Argumentos: tipo ("ingreso" o "gasto")
Retorna: Texto con la descripción o "No posee una descripción asignada"
pedirEsMensual(tipo): Confirma si el registro es mensual o no
Argumentos: tipo ("ingreso" o "gasto")
Retorna: true (si es mensual) o false (si no lo es)
pedirCategoria(tipo): Pregunta si desea agregar una categoría y permite elegir una de la lista
Argumentos: tipo ("ingreso" o "gasto").
Retorna: Nombre de la categoría seleccionada o "Varios"
pedirFecha(tipo): Pregunta si se usa la fecha actual o permite ingresar una en formato dd/mm/aaaa
Argumentos: tipo ("ingreso" o "gasto")
Retorna: Fecha en formato "dd/mm/aaaa"
pedirId(tipo): Pide un ID de registro existente para eliminarlo
Argumentos: tipo ("ingreso" o "gasto")
Retorna: Número del ID ingresado
ingresarRegistro(): Llama a las funciones anteriores para pedir datos, genera un ID y guarda el registro
Retorna: No retorna nada, pero agrega el registro a la lista correspondiente
mostrarRegistros(): Muestra todos los ingresos y gastos almacenados en la consola
Retorna: No retorna nada, solo imprime datos en la consola
eliminarRegistro(registro): Borra un registro de la lista según su ID
Argumentos: Objeto registro a eliminar
Retorna: No retorna nada, solo modifica la lista
buscarRegistro(ID, type): Busca un registro por su ID dentro de la lista de ingresos o gastos
Argumentos: ID (número) y type ("ingreso" o "gasto")
Retorna: Objeto registro encontrado o undefined si no existe
calcularSaldo(): Calcula la diferencia entre ingresos y gastos totales
Retorna: Número con el saldo actual
actualizarTotales(): Recalcula y actualiza los totales de ingresos, gastos y saldo en base a los registros almacenados.
Retorna: No retorna nada, solo actualiza los valores globales en la aplicación.
montoFormateoFloat(montoConFormato): Convierte un monto formateado en pesos argentinos a un número flotante.
Argumentos: montoConFormato (cadena de texto con formato de moneda)
Retorna: Número flotante con el valor del monto sin formato
montoFormateoPesos(montoSinFormato): Formatea un número a la notación de moneda en pesos argentinos.
Argumentos: montoSinFormato (número sin formato).
Retorna: Cadena de texto con el monto formateado en pesos argentinos

Funciones compuestas

ingresarRegistro(): Crea un nuevo registro de ingreso o gasto y lo almacena en la lista correspondiente.
Argumentos: Ninguno (solicita los datos al usuario mediante otras funciones).
Retorna: Objeto del registro creado.
mostrarRegistros(): Muestra en la consola todos los registros almacenados, diferenciando entre ingresos y gastos.
Argumentos: Ninguno.
Retorna: Nada (muestra los datos en la consola).
eliminarRegistro(registro): Elimina un registro de la lista según su ID.
Argumentos: Objeto registro (debe contener un ID válido).
Retorna: `true` si se eliminó correctamente, `false` si no se encontró el registro.
agregarRegistro(): Crea un nuevo registro de ingreso o gasto y lo almacena en la lista correspondiente.
Argumentos: No recibe argumentos directamente, pero usa los valores obtenidos de las funciones de entrada (tipo, monto, descripción, categoría, etc.).
Retorna: No retorna un valor explícito, pero actualiza la lista de registros almacenados.

Las instancias se crean mediante el método constructor que utiliza funciones basicas


Los datos se pierden al recargar la página. ⚠️ No hay almacenamiento permanente en esta versión
